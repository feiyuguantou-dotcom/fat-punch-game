## 一、产品调研依据与市场机会分析

### 1.1 市场调研数据支撑

#### 📊 小红书平台数据验证 (2025.7.1 - 2026.1.17)

| 核心发现 | 数据支撑 | 市场含义 |
|---------|---------|---------|

| **情绪宣泄需求巨大** | "发疯文学"爆款 **525,895赞+14,585评** | 用户需要"安全的发疯空间" |
| **解压品类成熟** | "解压小游戏"平均互动量 **3,200+** | 用户对解压游戏接受度高 |

### 1.2 市场机会分析

#### ✅ 为什么现在做这个产品?

1. **情感刚需 + 市场空白**
   - 失恋/分手是高频情感事件,每年有数千万用户经历
   - 现有解压游戏(捏泡泡、三消)缺乏情感共鸣
   - 没有专门针对"失恋情感"的游戏化产品

2. **微信生态红利**
   - 小程序即点即用,无需下载,适合情绪化场景
   - 社交关系链天然适合情感类内容传播
   - 女性用户占微信小程序核心用户60%+

3. **内容消费升级**
   - "发疯文学"流行,说明年轻用户需要情绪出口
   - 用户愿意为"情感疗愈"付费(心理APP、情感咨询等)
   - 游戏化产品比心理咨询更私密、更低门槛

4. **传播裂变潜力**
   - 小红书爆款笔记证明"前任话题"自带流量
   - 游戏化内容天然具备社交分享属性
   - 微信群/朋友圈是情感宣泄的高频场景

#### 🎯 目标市场规模估算

** TAM (总潜在市场)**:
- 中国每年失恋/分手人群: 约 3,000万+
- 情感类APP用户规模: 5,000万+
- 可触达用户: 1,000万+ (30%渗透率)

** SAM (可服务市场)**:
- 微信小程序月活用户: 4.5亿+
- 女性用户占比: 60%+
- 年龄18-35岁: 约1.5亿用户
- 情感内容关注用户: 约3,000万+

** SOM (可获得市场)**:
- 目标: 12个月内获得50万注册用户
- 月活用户目标: 15万+
- 付费转化率目标: 8-12%

## 二、产品定位与核心价值主张

### 2.1 产品定位

**一句话定位**:
> "一个可以安全地'揍前任'的微信小程序小游戏"

**产品Slogan**:
- 主标语: "失恋了?来揍前任吧!"
- 副标语: "打完就释怀,笑着向前看"

**产品类别**:
- 一级: 情感疗愈 / 心理健康
- 二级: 休闲小游戏 / 解压游戏
- 三级: 社交娱乐

### 2.2 核心价值主张

#### 对用户的3个核心价值:

1. **情感宣泄价值** - "安全的发疯空间"
   - 提供"不可言说"情绪的合法化出口
   - 避免真实伤害的虚拟化暴力表达
   - 即时、私密、无道德负担

2. **心理疗愈价值** - "游戏化的失恋自救"
   - 通过"仪式化"行为完成情感闭环
   - 从愤怒→发泄→释怀的心理过程引导
   - 获得掌控感和成就感

3. **社交连接价值** - "我不孤单"
   - 发现"原来大家都有这样的经历"
   - 共鸣内容的社交分享
   - 匿名社区的倾诉支持

### 2.3 差异化竞争优势

| **使用门槛** | ⭐⭐⭐⭐⭐ 即点即用 | ⭐⭐⭐⭐ 需下载 | ⭐⭐ 预约+付费 |

## 三、用户画像与使用场景

### 3.1 核心用户画像

#### 👩 主角画像: "失恋小美" (25岁,刚分手1周)

**基本信息**:
- 年龄: 25岁 (24-28岁核心)
- 性别: 女性 (占比85%+)
- 职业: 白领/学生/自由职业
- 收入: 月入5k-15k
- 地域: 一二线城市为主

**心理状态**:
- 情绪阶段: 愤怒/不甘/想报复 → 想忘忘不掉 → 释怀期
- 核心需求: "我想发泄,但不想真的伤害他"
- 痛点:
  - 夜间情绪反扑,不知道怎么办
  - 想骂他又怕显得自己不体面
  - 想报复但又怕承担后果

**行为特征**:
- 小红书重度用户,关注"情感"、"失恋"话题
- 喜欢刷"发疯文学"、"爽文"等内容
- 对"可爱"、"治愈"系内容无抵抗力
- 有一定付费意愿,但注重性价比

#### 🎮 次要用户画像

**围观群众** (35%):
- 未失恋但觉得好玩
- 纯粹为了体验游戏乐趣
- 可能成为传播节点

**男性用户** (15%):
- 被"渣男"视角吸引
- 好奇心理驱动
- 可能从"施暴者"角度体验

### 3.2 典型使用场景

#### 场景1: "深夜emo" (高频场景)
**时间**: 22:00 - 02:00
**触发**: 刷完小红书情感内容,情绪共鸣
**行为**: 打开小程序 → 选择"冷暴力前任" → 暴揍10次 → 看到释怀卡片 → 心情好转 → 睡觉

**产品诉求**:
- 操作简单(半梦半醒也能玩)
- 反馈即时(秒级解压)
- 无需思考(避免二次伤害)

#### 场景2: "闺蜜吐槽局" (社交场景)
**时间**: 周末下午茶
**触发**: 和闺蜜聊天聊到前男友
**行为**: 互相分享"你选哪个" → 展示战绩 → 比拼谁揍得更狠 → 发朋友圈

**产品诉求**:
- 分享战绩炫耀功能
- 闺蜜PK模式
- 朋友圈生成海报

#### 场景3: "刷到前任动态" (应激场景)
**时间**: 随时
**触发**: 刷到前任新动态/新恋情
**行为**: 紧急打开小程序 → 疯狂输出 → 情绪平复 → 继续生活

**产品诉求**:
- 秒级启动(快速加载)
- 连击模式(高频操作)
- 即时退出(随时停止)

#### 场景4: "分享给姐妹" (传播场景)
**时间**: 使用后
**触发**: 觉得解压/好玩
**行为**: 分享到微信群 → 生成"渣男类型测试" → 姐妹们跟玩 → 裂变传播

**产品诉求**:
- 社交分享 incentives
- 性格测试功能
- 邀请机制设计

## 四、产品功能需求

### 4.1 核心功能架构 (MVP版本)

#### 🎮 游戏模式

##### 模式A: 单人挑战模式 (核心模式)

**游戏流程**:
```
1. 选择渣男类型 (30秒)
   ↓
2. 暴揍环节 (3-5分钟)
   ↓
3. 结局解锁 (情感闭环)
   ↓
4. 社交分享 (可选)
```

**步骤1: 选择渣男类型** (角色定制)
- 提供6种预设渣男类型:
  1. 💀 "冷暴力渣男" - 不回消息、忽冷忽热
  2. 🔥 "出轨渣男" - 精神/肉体出轨
  3. 💰 "抠门渣男" - 一毛不拔、精打细算
  4. 🎭 "撒谎渣男" - 满嘴跑火车
  5. 👻 "妈宝渣男" - 什么都听妈妈的
  6. 💔 "无缝衔接渣男" - 分手立刻有新欢

- **交互方式**: 横向滑动卡片选择
- **交互细节**:
  - 每个类型配标志性台词(如冷暴力渣男:"我忙")
  - 点击卡片出现"简介弹窗"(3句话描述)
  - 支持自定义渣男(可选功能:输入名字+上传头像)

**步骤2: 暴揍环节** (核心玩法)

**基础攻击方式**:
1. **点击攻击** - 主要交互方式
   - 单点: 一次攻击 -1血量
   - 连点: 快速连击,触发连击特效
   - 长按: 蓄力大招,一次-10血量

2. **技能攻击** (付费/解锁)
   - 技能1: "闪电五连鞭" - 连续5次暴击
   - 技能2: "真心话炸弹" - 随机扎心语录
   - 技能3: "变形术" - 渣男变青蛙/猪/癞蛤蟆

**进度反馈系统**:
- **血条显示**: 渣男头顶血条,实时更新
- **连击Combo**: 连击3次触发"暴击!"
- **等级进度**:
  - Lv1: 陌生路人 (0-20血)
  - Lv2: 认识的人 (21-50血)
  - Lv3: 前男友 (51-100血)
  - Lv4: 前男友(被揍版) (101-150血)

**音效设计**:
- 普通攻击: "啪"、"duang"、"咚"
- 暴击: "砰!"、"裂开!"
- 变形: "biu~" (变身音效)
- 胜利: "你值得更好的!" (温柔女声)

**步骤3: 结局解锁** (情感闭环)

**基础结局** (免费):
- 渣男道歉视频 (卡通动画)
  - 台词: "对不起,是我不好了"
  - 表情: 懊悔、流泪
- 释怀卡片生成
  - 文案: "你已经放下啦,前方有更好的人等你"
  - 配图: 温暖插画 (彩虹、阳光)

**高级结局** (付费/分享解锁):
- "真爱测试" - 告诉你什么时候会遇到对的人
- "报应预警" - 前任什么时候会遭报应
- "桃花运分析" - 你的恋爱运势

**步骤4: 社交分享** (裂变核心)

**分享触发点**:
- 游戏结束页面
- 达成特殊成就
- 解锁高级结局

**分享内容**:
- 分享到微信群: "我暴揍了XX次渣男,超解压!你也来试试"
- 分享到朋友圈: 生成战绩海报
  - 主文案: "分手第X天,我终于释怀了"
  - 副文案: "你也来揍前任吧!"
  - 图片: 游戏截图 + 装饰元素

**分享激励**:
- 分享1次: 解锁新攻击特效
- 分享3次: 解锁新渣男类型
- 邀请1人: 获得"复仇基金"(红包)

##### 模式B: 性格测试模式 (引流模式)

**玩法流程**:
```
1. 回答5个问题 (1分钟)
   ↓
2. 生成"你的渣男类型"报告 (30秒生成)
   ↓
3. 分享测试结果 (社交裂变)
   ↓
4. 引导到游戏 (转化)
```

**测试问题设计**:
1. "你的前任最常说的是?"
   A. "我忙"  B. "你无理取闹"  C. "都是你的错"
2. "分手时他的反应是?"
   A. 冷暴力  B. 立刻有新欢  C. 痛哭流涕
3. "你最想对他说什么?"
   A. "滚!"  B. "祝你幸福"  C. "为什么?"
4. "如果有机会,你会?"
   A. 揍他  B. 骂他  C. 无视他
5. "你现在的心情是?"
   A. 愤怒  B. 不甘心  C. 释怀

**报告生成**:
- 渣男类型判定 (冷暴力型/出轨型/妈宝型...)
- 测试者性格分析 (复仇女神/温柔小兔/理性女王...)
- 匹配度分析 (你值得什么样的人)
- 行动建议 (推荐游戏模式)

**社交裂变**:
- 分享到朋友圈:
  - 标题: "测测你的渣男类型,超准!"
  - 配图: 测试结果卡片
- 分享到微信群:
  - 引导: "来看看你的姐妹是什么类型"

#### 📊 数据统计后台 (后台管理)

**核心指标**:
- DAU/MAU
- 游戏完成率
- 平均游戏时长
- 分享率
- 付费转化率
- ARPU (每用户平均收入)
- 留存率

**用户行为分析**:
- 渣男类型选择分布
- 攻击方式偏好
- 结局选择偏好
- 社交分享路径

### 6.2 技术选型理由

#### 为什么选择微信小程序?

1. **用户获取成本最低**
   - 无需下载,即点即用
   - 朋友圈/微信群天然传播渠道
   - 扫码即可体验,降低获客门槛

2. **开发成本可控**
   - 相比APP,小程序开发周期短
   - 云开发模式,运维成本低
   - 微信提供丰富的API支持

3. **社交裂变最佳载体**
   - 群消息分享无缝集成
   - 朋友圈分享自带信任背书
   - 关系链传播效率最高

4. **政策风险较低**
   - 游戏化情感疗愈,定位清晰
   - 不涉及真实暴力,内容安全可控
   - 符合微信小程序审核规范

exports.reportEvent = async (event) => {
  const { eventName, eventData } = event;

  // 1. 数据验证
  if (!eventName || !eventData) {
    return { error: '参数错误' };
  }

  // 2. 记录基础信息
  const logData = {
    eventName,
    eventData,
    timestamp: new Date(),
    userId: eventData.userId || '',
    openId: eventData.openId || '',
    ...eventData
  };

  // 3. 写入数据库
  await db.collection('analytics').add(logData);

  // 4. 实时统计 (可选)
  await updateRealtimeStats(eventName);

  return { success: true };
};

### 11.1 内容安全

**过滤机制**:
1. 用户输入实时过滤

3. AI + 人工双重审核

**审核机制**:
- 自动审核: 微信内容安全API

- 用户举报: 接收用户举报并处理

**违规处理**:
- 警告: 首次违规,内容下架

- 必需数据: openId (唯一标识)、昵称、头像

#### 数据使用规范

**使用原则**:
1. 仅用于产品功能优化
2. 不向第三方出售
3. 不用于个性化广告 (微信小程序规范)
4. 数据加密存储和传输

#### 用户权利

**用户权利**:
- 访问权: 查看个人数据

- 删除权: 注销账号并删除数据

### 11.3 防作弊机制

#### 刷行为识别

**作弊行为**:
- 刷脚本攻击
- 刷评论
- 刷分享

**防范措施**:
- 频率限制: 单用户操作频率限制

- 用户社区运营
- 内容创作能力

### 13.3 技术风险

**风险**: 服务器崩溃、加载缓慢

**应对措施**:
- 云服务器弹性扩容
- CDN加速
- 资源预加载
- 降级策略

**风险**: 数据泄露

**应对措施**:
- 数据加密存储
- 定期安全审计
- 最小化数据收集
- 用户数据匿名化

### 13.4 运营风险

**风险**: 用户增长不及预期

**应对措施**:
- 多渠道获客,避免单一渠道依赖
- 优化产品功能,提升用户粘性
- 加强内容运营,提高分享率
- 调整运营策略,灵活应对

**风险**: 负面口碑

**应对措施**:
- 及时回应用户反馈
- 快速修复产品问题
- 公开透明处理负面事件
- 持续改进产品体验
